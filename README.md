# quiz-game

Мультиплеерная браузерная квиз-викторина в стиле Kahoot. Один человек создаёт
комнату как ведущий, остальные (2-8 человек) подключаются с любого устройства
по коду комнаты и отвечают на вопросы в реальном времени.

## Как это работает

1. Ведущий открывает `/host.html`, нажимает «Создать комнату» — получает
   4-буквенный код.
2. Игроки открывают `/player.html`, вводят имя и код комнаты.
3. Ведущий выбирает тему викторины (см. [Пакеты вопросов](#пакеты-вопросов))
   и нажимает «Начать игру» — всем рассылается первый вопрос с таймером
   (15 секунд).
4. Игроки выбирают вариант ответа. За правильный ответ начисляется от 10 до
   20 очков (бонус за скорость).
5. После ответа всех игроков (или истечения таймера) показываются результаты
   раунда, затем — следующий вопрос. После последнего вопроса — финальная
   таблица лидеров.

## Пакеты вопросов

Вопросы разбиты по темам в `packs/*.json`: «Общие знания», «Наука»,
«История», «Кино и видеоигры», «Спорт». Ведущий выбирает тему в лобби перед
стартом игры. Каждый файл пакета имеет формат:

```json
{
  "topic": "Название темы",
  "questions": [
    { "question": "...", "options": ["...", "...", "...", "..."], "correctIndex": 0 }
  ]
}
```

Чтобы добавить новую тему, создайте `packs/<id>.json` в этом формате —
сервер подхватывает все файлы из `packs/` при старте.

Часть контента в пакетах «Наука», «История», «Кино и видеоигры», «Спорт»
была сгенерирована черновиком через локальную модель Ollama (см.
[ollama-mcp](https://github.com/KoDiK2005/ollama-mcp)) и затем вручную
проверена/исправлена — у небольшой локальной модели находились фактические
ошибки (перепутанные даты, несуществующие персонажи), так что черновой
контент от локальной модели не стоит использовать без проверки.

## Стек

- Сервер: Node.js, Express, Socket.io (комнаты, синхронизация состояния игры)
- Клиент: чистый HTML/CSS/JS, без сборки

## Установка и запуск

```bash
cd quiz-game
npm install
npm start
```

Сервер поднимется на `http://localhost:3000`.

- Ведущий открывает `http://localhost:3000/host.html`
- Игроки открывают `http://localhost:3000/player.html` (с других устройств в
  той же сети — по IP компьютера-сервера, например `http://192.168.x.x:3000/player.html`)

## Структура проекта

```
quiz-game/
├── server.js          # Express + Socket.io сервер, логика комнат и раундов
├── lib/gameLogic.js    # Чистые функции: код комнаты, очки, таблица лидеров
├── packs/*.json         # Пакеты вопросов по темам
├── test/                # Jest-тесты + ручной end-to-end smoke-тест
├── package.json
└── public/
    ├── index.html       # Лендинг с выбором роли
    ├── host.html/.js     # Панель ведущего (выбор темы, старт, результаты)
    ├── player.html/.js   # Экран игрока
    └── style.css
```

## Тесты

```bash
npm test
```

Юнит-тесты покрывают `lib/gameLogic.js` (генерация кода комнаты, начисление
очков, сортировка таблицы лидеров). Дополнительно есть ручной
end-to-end smoke-тест, поднимающий настоящий сервер и эмулирующий хоста и
игрока через `socket.io-client`:

```bash
node test/smoke.manual.js
```
